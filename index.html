👥 客戶管理

每位客戶對應一個 GitHub 私人倉庫存放資料。切換客戶時自動上傳當前資料、再下載新客戶資料。

📘 新增客戶的步驟（點開展開） 
1. 到 https://github.com/new 建立新的私人倉庫
   名稱建議：abc-tracker-data-客戶名（例：abc-tracker-data-王小明）
   勾選 Private、Add a README file
2. 確保您的 Token 有訪問此新倉庫的權限：
   到 https://github.com/settings/personal-access-tokens
   點您現有的 token (abc-tracker) → 編輯 → Repository access
   勾選新建立的倉庫 → Update
3. 回到下方表單，按「＋ 新增客戶」填入：
   客戶名稱（顯示用，例：王小明）
   倉庫名稱（剛建立的，例：abc-tracker-data-王小明）
   檔案路徑（保持預設 data/state.json）
4. 按「儲存」即可
5. 之後在頂端切換器下拉選單就能切換客戶

🐙 GitHub 私人倉庫自動同步

目前狀態：未啟用

📘 第一次設定步驟（點開展開） 
步驟 1. 建立私人倉庫
  ① 到 https://github.com/new
  ② Repository name：填一個容易記的（例：abc-tracker-data）
  ③ 選「Private」（私人）
  ④ 勾「Add a README file」→ 按「Create repository」

步驟 2. 取得 Personal Access Token（fine-grained，最安全）
  ① 到 https://github.com/settings/personal-access-tokens/new
  ② Token name：abc-tracker
  ③ Expiration：選 1 年（到期後再換）
  ④ Repository access：「Only select repositories」→ 選剛建立的倉庫
  ⑤ Permissions → Repository permissions
       找到「Contents」→ 改成「Read and write」
  ⑥ 按下方「Generate token」→ 把出現的 token 複製起來
     ⚠️ Token 只會顯示一次，請立刻貼到下面框框

步驟 3. 填入下方欄位後按「儲存並連線」

⚠️ 安全提醒
  • Token 等於倉庫的鑰匙，會存在您瀏覽器 localStorage
  • 請勿在公用電腦使用
  • 設定 fine-grained token 只給該倉庫權限，比 classic token 安全
  • Token 不要分享給別人
